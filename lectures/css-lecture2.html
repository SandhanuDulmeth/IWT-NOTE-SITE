CSS Preprocessors & Modern CSS

Limitations of traditional CSS, preprocessors (Sass, Less, Stylus), modern CSS features including CSS variables, and attribute selectors.

01 Limitations & Preprocessors

Lecture 2 ¬∑ SCS 1311 ¬∑ Topics: Traditional CSS problems, preprocessor concept

‚ö†Ô∏è Limitations of Traditional CSS

  • Large stylesheets were hard to maintain
  • Code repetition was common
  • Difficult to reuse values like colors or fonts
  • No logic support (conditions, loops)
  • Flat structure with poor organization

üîß What is a CSS Preprocessor?

A CSS preprocessor is a scripting language that extends CSS.

  1. You write code in the preprocessor language
  2. It compiles into normal CSS
  3. Browser reads the compiled CSS

‚ú® Common Features of Preprocessors

  • Variables ‚Äî store reusable values
  • Nesting ‚Äî reflect HTML structure in CSS
  • Mixins ‚Äî reusable code blocks
  • Functions ‚Äî dynamic calculations
  • Mathematical operations
  • Better code organization

üèÜ Popular CSS Preprocessors

Preprocessor Variable Syntax Notes
Sass $variable Most popular, feature-rich
Less @variable Beginner-friendly, used in Bootstrap v3
Stylus variable = Flexible syntax, less common today

02 Sass Preprocessor

üé® Sass Variables

  • Used to store reusable values
  • Variables start with $
  • Defined once, reused many times
  • Browser never sees the variable ‚Äî Sass replaces it during compilation
scss ‚Üí css 
/* Sass (SCSS) */
$main-color: #0066cc;

.message {
  color: $main-color;
}

/* Compiled CSS */
.message {
  color: #0066cc;
}

ü™∫ Sass Nesting

  • Nesting reflects the HTML structure inside CSS
  • Rules are nested inside parent selectors
  • Sass converts them into flat CSS
scss ‚Üí css 
/* Sass (SCSS) */
nav {
  ul {
    list-style: none;
  }
  li {
    display: inline-block;
  }
}

/* Compiled CSS */
nav ul { list-style: none; }
nav li { display: inline-block; }

üîÑ Sass Mixins

  • Mixins are reusable blocks of CSS
  • Defined using @mixin
  • Used with @include
  • ‚ö†Ô∏è Overuse may increase CSS output size
scss ‚Üí css 
/* Sass (SCSS) */
@mixin box {
  padding: 10px;
  border: 1px solid #ccc;
}

.message {
  @include box;
}

/* Compiled CSS */
.message {
  padding: 10px;
  border: 1px solid #ccc;
}

üß¨ Sass Inheritance

  • Allows one selector to reuse styles from another
  • Uses @extend
  • Sass merges selectors
  • Not true object-oriented inheritance
scss 
.success {
  @extend .message;
  color: green;
}

üì• Sass Imports

  • Split styles into multiple files
  • Combines them into one CSS file on compilation
  • Browser loads only one CSS file
  • Modern Sass prefers @use instead of @import
scss 
@import "buttons";

üèãÔ∏è Practice

  1. Create Sass variables for primary color, font family, and spacing
  2. Write a nested Sass rule for a navigation with ul, li, a
  3. Create a mixin for a card component with padding, border, and shadow
  4. Use @extend to share base styles between .btn and .btn-primary

03 Less & Stylus

üíé Less (Leaner Style Sheets)

  • Syntax close to CSS ‚Äî beginner-friendly
  • Variables use @
  • Used in older projects (e.g., Bootstrap v3)
less 
@color: #333;

body {
  color: @color;
}

üñåÔ∏è Stylus

  • Very flexible syntax
  • Can omit braces, colons, semicolons
  • Variables use =
  • Supports conditionals and logic
  • Less common today
stylus 
color = #333

body
  color color

‚úÖ Advantages of Preprocessors

  • Cleaner, more maintainable code
  • Better organization
  • Faster team development
  • Used in many frameworks

‚ùå Disadvantages of Preprocessors

  • Need extra tools
  • Compilation step required
  • Learning curve
  • Many features now exist in plain CSS

04 Modern CSS

üöÄ Why Preprocessors Are Less Needed Today

Modern CSS includes features that once required preprocessors:

  • CSS variables (Custom Properties)
  • Native nesting
  • calc() and other functions
  • Better browser support

Extra build steps, maintenance complexity, and a fragmented ecosystem make preprocessors less attractive when native CSS can do the job.

üéØ CSS Variables (Custom Properties)

  • Declared with -- prefix
  • Used with var() function
  • Can be scoped to any selector (commonly on :root)
  • Browser understands them natively ‚Äî no compilation needed
css 
:root {
  --main-color: #333;
}

p {
  color: var(--main-color);
}

üí° Big Picture ‚Äî Preprocessors vs Modern CSS

Preprocessors were like exoskeletons for CSS ‚Äî external tools that gave it muscles, memory, and logic.

Modern CSS is slowly evolving those features inside its own body, like an organism growing new organs. The ecosystem is shifting from "compile before using" to "use directly in the browser."

Technology often does this: first we build tools around a language, then the language absorbs the best ideas and becomes stronger.

üèãÔ∏è Practice

  1. Define CSS custom properties for --primary, --secondary, and --font-size on :root
  2. Use var() to apply those variables across multiple elements
  3. Compare writing the same component in Sass vs plain CSS variables

05 Attribute Selectors

üè∑Ô∏è What Are Attribute Selectors?

Select elements based on their attributes. Useful when:

  • No class or ID available
  • Styling depends on attribute value
  • Targeting forms, links, metadata

üìã Types of Attribute Selectors

Selector Meaning Example
[attr] Has the attribute input[required]
[attr=val] Exact match input[type="email"]
[attr*=val] Contains value a[href*="google"]
[attr^=val] Starts with a[href^="https"]
[attr$=val] Ends with a[href$=".pdf"]
[attr~=val] Word in list p[class~="highlight"]
[attr|=val] Exact or hyphenated p[lang|="en"]
css 
/* Has attribute */
input[required] {
  border: 2px solid red;
}

/* Exact match */
input[type="email"] {
  background-color: #f0f8ff;
}

/* Contains */
a[href*="google"] {
  color: green;
}

/* Starts with */
a[href^="https"] {
  font-weight: bold;
}

/* Ends with */
a[href$=".pdf"] {
  color: red;
}

/* Word in space-separated list */
p[class~="highlight"] {
  background-color: yellow;
}

/* Exact or hyphenated */
p[lang|="en"] {
  color: blue;
}

üèãÔ∏è Practice

  1. Style all input[required] fields with a red border
  2. Use [type="email"] to give email fields a unique background
  3. Highlight all external links using [href^="https"]
  4. Style PDF links differently using [href$=".pdf"]
  5. Use [lang|="en"] to target English-language paragraphs
