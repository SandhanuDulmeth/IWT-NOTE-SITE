Layouts & Responsive Web Design

CSS display property, the box model, Flexbox layout, responsive design principles, and media queries for building modern, adaptive web pages.

01 Display Property & Box Model

Lecture 3 ¬∑ SCS 1311 ¬∑ Topics: Layout importance, display values, block vs inline, box model

üìê Why CSS Layout Matters

  • Styling individual elements is not enough
  • Real websites contain headers, navs, content areas, sidebars, footers
  • Layout controls where elements appear on the screen

üìú Layout Evolution

  • Early web: tables and float-based layouts (complex, fragile)
  • Modern CSS: Flexbox (1D) and CSS Grid (2D)

üñ•Ô∏è CSS display Property

Controls how an element is rendered.

Value Behaviour
block Full width, starts on new line
inline Only required width, same line
inline-block Same line but accepts width/height
none Element is hidden

üß± Block vs Inline Elements

Block Inline
Starts on new line Stays on the same line
Full available width Only required width
div, p, h1 span, a, strong

Inline-block ‚Äî appears on the same line like inline but allows width & height like block. Useful for nav menus and small card layouts.

üì¶ The CSS Box Model

Every HTML element is a rectangular box with four layers:

  1. Content ‚Äî actual text or image (controlled by width / height)
  2. Padding ‚Äî space inside the border
  3. Border ‚Äî surrounds the padding
  4. Margin ‚Äî space outside the element

Understanding the box model helps avoid unexpected spacing and sizing issues.

üèãÔ∏è Practice

  1. Change a <span> to display: block and observe the difference
  2. Create three inline-block boxes side by side
  3. Add padding, border, and margin to an element and inspect the box model in DevTools

02 Flexbox Basics

üí™ What is Flexbox?

A modern CSS layout system used to:

  • Align elements easily
  • Distribute space efficiently
  • Adapt to different screen sizes

Flexbox works in one direction at a time:

  • Horizontal ‚Üí row
  • Vertical ‚Üí column

üî≤ Flex Container & Flex Items

  • Parent becomes the flex container
  • Children become flex items
  • Only direct children are affected
css 
.wrapper {
  display: flex;
}

‚û°Ô∏è Flex Direction

Defines the direction of items inside the container.

Value Direction
row (default) Left ‚Üí Right
column Top ‚Üí Bottom
css 
.container {
  display: flex;
  flex-direction: column;
}

üèãÔ∏è Practice

  1. Create a flex container with three child boxes
  2. Switch between flex-direction: row and column
  3. Observe how only direct children become flex items

03 Flex Properties

‚ÜîÔ∏è justify-content (Main Axis)

Controls alignment along the main axis ‚Äî used to center items, add space between them, or distribute them evenly.

Value Effect
flex-start (default) Pack items at the start
flex-end Pack items at the end
center Center items
space-between Equal space between, none at edges
space-around Equal space around each item
space-evenly Equal space everywhere
css 
.container {
  display: flex;
  justify-content: space-between;
}

‚ÜïÔ∏è align-items (Cross Axis)

Controls alignment along the cross axis ‚Äî used for vertical centering and aligning items of different heights.

Value Effect
stretch (default) Items stretch to fill container
flex-start Align to top
flex-end Align to bottom
center Center vertically
baseline Align by text baseline
css 
.container {
  display: flex;
  align-items: center;
}

üìè gap Property

Defines space between flex items ‚Äî cleaner than margins and easier to maintain.

Value Example
gap: 0; No gap
gap: 10px; Fixed pixel gap
gap: 1rem; Relative to root font size
gap: 5%; Percentage of container
gap: 10px 20px; Row gap & column gap
css 
.container {
  display: flex;
  gap: 15px;
}

üèãÔ∏è Practice

  1. Try every justify-content value and observe the differences
  2. Vertically center items using align-items: center
  3. Replace margins between items with the gap property

04 Page Layout with Flexbox

üèóÔ∏è Typical Page Layout

  • Header
  • Main content + Sidebar
  • Footer

Flexbox allows automatic adjustment based on screen size.

css 
.page {
  display: flex;
}

main {
  flex: 3;   /* takes 3 parts of space */
}

aside {
  flex: 1;   /* takes 1 part of space */
}

ü™Ü Nested Flexboxes

  • Flex layouts can exist inside other flex layouts
  • Example: page uses Flexbox, navigation bar inside it also uses Flexbox
  • Allows fine-grained control over every section

üèãÔ∏è Practice

  1. Build a page with header, main (3/4 width), sidebar (1/4 width), and footer
  2. Use flex: 3 and flex: 1 to control proportions
  3. Nest a Flexbox nav inside the header

05 Responsive Web Design

üì± What is Responsive Design?

Ensures a site works on desktops, tablets, and mobile phones. Same HTML adapts visually using CSS.

‚ùì Why It Matters

  • Most users browse on mobile
  • Devices have different screen sizes
  • Users expect: no zooming, no horizontal scrolling

‚öñÔ∏è Non-Responsive vs Responsive

Non-Responsive Responsive
Fixed width layout Layout adjusts automatically
Only for desktop Works on all devices
Horizontal scrolling Content stacks vertically
Poor UX on mobile Easy to read & navigate

üñºÔ∏è Responsive Images

Images should scale with their container:

css 
img {
  max-width: 100%;
  height: auto;
}

‚ö†Ô∏è Common Layout Mistakes

  • Using fixed widths everywhere
  • Ignoring mobile users
  • Mixing outdated layout techniques
  • Not testing on different screen sizes

06 Media Queries

üìê What Are Media Queries?

Apply CSS only under certain conditions. Used to:

  • Change layouts on smaller screens
  • Adjust font sizes
  • Rearrange content
css 
/* Desktop layout */
.container {
  display: flex;
  gap: 10px;
}

.box {
  width: 50%;
}

/* Mobile layout */
@media (max-width: 768px) {
  .container {
    flex-direction: column;
  }

  .box {
    width: 100%;
  }
}

üèãÔ∏è Practice

  1. Create a two-column layout that stacks into one column below 768px
  2. Change the font size of headings at different breakpoints
  3. Make all images responsive with max-width: 100%
  4. Test your layout at 1200px, 768px, and 480px widths
