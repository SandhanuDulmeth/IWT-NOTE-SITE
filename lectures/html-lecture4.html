04 Links & Navigation

Lecture 4 ¬∑ SCS 1311 ¬∑ Topics: Hyperlinks, file paths, bookmarks, image maps

üîó Hyperlinks

  • Connects one web page to another ‚Äî allows navigation
  • Created using the <a> (anchor) tag
  • Syntax: <a href="url">Link text</a>
Internal Links
  • Link to another page in the same website ‚Äî use relative paths
  • Example: <a href="about.html">About Us</a>
External Links
  • Link to another website ‚Äî use absolute URLs
  • Example: <a href="https://www.ucsc.cmb.ac.lk">UCSC</a>

üìÇ HTML File Paths

  • A path tells the browser where a file is located
  • Used in: href, src, action
Type Description Example
Absolute (External) Full URL including protocol https://www.google.com
Root-Relative Starts from root folder with / /pages/contact.html
Relative Relative to the current file pages/contact.html
Special symbols . = current folder ¬∑ .. = parent folder ../images/logo.png

üîñ Bookmarks (Page Anchors)

  • Jump to a specific part of a page
  • id ‚Üí sets the target ¬∑ # ‚Üí links to it
  • Same page: <a href="#contact">Go to Contact</a>
  • Another page: <a href="about.html#team">Our Team</a>

üó∫Ô∏è Image Mapping

  • Different parts of one image act as separate links
  • Uses: <img usemap> + <map> + <area>
  • Shapes: rect (rectangle), circle, poly (polygon)
  • Coordinates measured from top-left corner
html 
<!-- Internal link -->
<a href="about.html">About Us</a>

<!-- External link -->
<a href="https://www.ucsc.cmb.ac.lk">UCSC Website</a>

<!-- File paths -->
<a href="pages/contact.html">Contact</a>       <!-- relative -->
<a href="/pages/contact.html">Contact</a>      <!-- root-relative -->
<!-- Reference to parent folder (WE USE favicon.svg HERE AS IT EXISTS) -->
<img src="../images/favicon.svg" width="50">

<!-- Bookmark target -->
<h2 id="contact">Contact</h2>
<!-- Link to bookmark -->
<a href="#contact">Go to Contact</a>
<a href="about.html#team">Our Team</a>

<!-- Image map -->
<img src="https://placehold.co/400x300?text=Park+Map" usemap="#parkmap">
<map name="parkmap">
  <area shape="rect" coords="60,60,180,150" href="playground.html" alt="Playground">
  <area shape="circle" coords="300,200,50" href="fountain.html" alt="Fountain">
</map>

üí° Conceptual: Website as a City

  • Folders ‚Üí neighbourhoods ¬∑ Files ‚Üí houses
  • Paths ‚Üí directions ¬∑ Hyperlinks ‚Üí roads
  • The browser is a courier ‚Äî give it a path, it walks the folders step by step
  • Understanding relative vs absolute paths is one of the most practical web skills

üèãÔ∏è Practice

  1. Create internal links between 3 HTML pages in the same folder
  2. Add an external link to a website with target="_blank"
  3. Use .. to link to a file in a parent directory
  4. Create bookmark links to jump between sections on the same page
  5. Build an image map with at least 2 clickable areas

05 Forms & Input

Lecture 4 (continued) ¬∑ Topics: Form attributes, elements, input types, GET vs POST

üìã HTML Forms

  • Forms allow users to send data to a server
  • Created with <form>
  • Can contain: text fields, checkboxes, radio buttons, dropdowns, buttons

‚öôÔ∏è Form Attributes

Attribute Purpose
action Where form data is sent
method How data is sent (GET or POST)
target Where to display response (_self, _blank, _parent, _top)
enctype Encoding type for form data
novalidate Disables form validation
autocomplete Enables/disables autofill (on/off)
Attribute Examples 
<!-- Open response in a new tab -->
<form action="/search" target="_blank">

<!-- Turn off autocomplete for sensitive data -->
<form action="/login" autocomplete="off">

<!-- Required for file uploads -->
<form enctype="multipart/form-data" method="post">
  <input type="file" name="doc">
</form>

<!-- Submit without validating data -->
<form novalidate>
  <input type="email" name="email">
</form>

üß© Form Elements

Element Purpose
<input> Various input types (text, email, password‚Ä¶)
<label> Labels for inputs ‚Äî for links to input id
<button> Clickable button (submit, reset, button)
<select> Dropdown list (with <option> and <optgroup>)
<textarea> Multi-line text input (rows / cols)
<fieldset> + <legend> Groups related form elements with a caption
<datalist> Suggestion list for input (<input list="id">)
<output> Displays a calculation result

‚å®Ô∏è Input Types

Type Description
text Single-line text
password Masked text
email Email with validation
number Numeric input
date / time / month / week Date and time pickers
checkbox Multiple selections
radio Single selection per group (same name)
file File upload
color Colour picker
range Slider (min / max)
url / tel / search Specialised text inputs
hidden Invisible data field
submit / reset / button Form action buttons

üì° GET vs POST

GET POST
Data visible in URL Data in request body
Default method Must be specified
Less secure More secure
