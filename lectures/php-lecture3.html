Forms, Validation & MySQL

Understanding HTML forms and PHP flow, PHP Superglobals, GET/POST methods, form validation, and connecting PHP to MySQL databases.

01 HTML Forms & PHP Flow

Lecture 3 ¬∑ SCS 1311 ¬∑ Topics: forms, flow, superglobals

üìù What is an HTML Form?

  • An HTML form collects user input.
  • When submitted, data is sent to the server.
  • PHP receives and processes this data.
  • The server sends a response back to the user.

User ‚Üí HTML Form ‚Üí PHP Script ‚Üí Response

üåê PHP Superglobals

Superglobals are built-in PHP arrays that are accessible anywhere in a script without needing to declare them global.

üìå Common Superglobals:

  • $_GET ‚Äî for URL parameters
  • $_POST ‚Äî for HTTP POST data
  • $_REQUEST ‚Äî contains both GET, POST, and COOKIE data

02 PHP Methods

üîç GET Method

  • Data sent through URL (query string).
  • Visible in browser address bar (?name=value).
  • Can be bookmarked or shared.
  • Not suitable for sensitive information.

üîí POST Method

  • Data sent in HTTP request body.
  • Not visible in URL.
  • Cannot be bookmarked.
  • Suitable for: Sensitive data (passwords) and Large submissions.
php ‚Äî $_GET and $_POST 
// $_GET collects values sent with method="get"
$_GET['username'];  // Keys are form field names

// $_POST collects values sent with method="post"
$_POST['username'];  // Values are user inputs

03 Handling Data

html ‚Äî Basic HTML Form 
<form method="post" action="">
 <label for="sport">Enter your favorite sport:</label><br><br>

 <input type="text" id="sport" name="sport"><br><br>

 <input type="submit" name="submit" value="Submit">
</form>

‚öôÔ∏è Understanding the action Attribute

  • action="" ‚Üí submits back to the same PHP file
  • action="process.php" ‚Üí sends data to another file

üìå Why use an explicit action (different file)?

Provides clear application flow, is better for large projects, and allows for easier maintenance (Separate Processing File Pattern).

php ‚Äî Same-Page Form Handling Example 
// File: index.php
<?php
$sport = "";

// Check if form is submitted via POST
if (isset($_POST["submit"])) {
  $sport = $_POST["sport"];  // Retrieve value
}
?>

<!-- Form HTML goes here -->

<?php
if ($sport != "") {
  echo "Your favorite sport is: " . $sport;
}
?>
html/php ‚Äî Other Form Elements (Radio & Checkbox) 
<!-- Radio Buttons -->
<input type="radio" name="sport" value="Soccer"> Soccer
<input type="radio" name="sport" value="Cricket"> Cricket

<?php
  // Access radio button value

<!-- Checkboxes (returns array) -->
<input type="checkbox" name="sports[]" value="Soccer"> Soccer

<?php

  $sports = $_POST["sports"];

  }
?>

04 Form Validation & Sanitization

üõ°Ô∏è Why Validation?

  • Prevent empty/wrong data.
  • Prevent errors and application crashes.
  • Improve User Experience (UX).
  • Essential for security to stop malicious data.
php ‚Äî Validation & Retaining Input 
<?php
$error = "";
$sport = "";

if (isset($_POST["submit"])) {
  // Using empty() to check empty input
  if (empty($_POST["sport"])) {
    $error = "Sport is required";
  } else {
    // Sanitization to prevent HTML injection and prevent scripts
    $sport = htmlspecialchars($_POST["sport"]);
  }
}
?>

<!-- Displaying Validation Errors -->
<span style="color:red;"><?php echo $error; ?></span>

<!-- Retaining Input Values (prevents retyping after submit) -->
<input type="text" name="sport" value="<?php echo $sport; ?>">

05 Introduction to Databases

‚ùì Why Do We Need a Database?

When PHP finishes running:

  • Variables disappear
  • Data is not saved
  • Sessions are temporary
  • Files are not structured for large storage

üíæ Databases solve this by letting us:

Store users, login details, file names, retrieve data later, and update/delete records easily.

üê¨ What is MySQL?

  • A Relational Database Management System (RDBMS)
  • Uses SQL (Structured Query Language)
  • Stores data in tables containing rows & columns
  • PHP interacts with MySQL via SQL queries

üîë SQL Key Concepts

  • Database: container for tables.
  • Table: collection of related data entries.
  • Row: a single record in a table.
  • Column: a specific field (e.g., age, name).
  • Primary Key: unique identifier for each row.

üî† MySQL Data Types

  • Numeric: INT, DECIMAL(10,2), FLOAT
  • String: VARCHAR(255), TEXT
  • Date/Time: DATE, DATETIME, TIMESTAMP

06 MySQL Integration

sql ‚Äî Creating Database & Table 
-- Creating Database
CREATE DATABASE school_db;

-- Creating Table
CREATE TABLE students (
 id INT AUTO_INCREMENT PRIMARY KEY,
 name VARCHAR(100) NOT NULL,
 email VARCHAR(100) UNIQUE NOT NULL,
 age INT,
 created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

üåê Connecting PHP to MySQL

Three common methods are: 1) MySQLi (Procedural), 2) MySQLi (OOP), and 3) PDO (PHP Data Objects).

php ‚Äî MySQLi OOP Connection 
<?php
$servername = "localhost";
$username = "root";
$password = "";
$database = "school_db";

// Create connection
$conn = new mysqli($servername, $username, $password, $database);

// Check connection
if ($conn->connect_error) {
  die("Connection failed: " . $conn->connect_error);
}

echo "Connected Successfully";

// Good practice: Only keep connection code in a dedicated file (db.php)
// Closing the connection
$conn->close();
?>

üß© Important MySQLi Concepts

  • mysqli ‚Üí built-in PHP class
  • $conn ‚Üí connection object
  • $conn->connect_error ‚Üí stores error if failed
  • die() ‚Üí stops script execution
  • $conn->query() ‚Üí run queries
  • $result->fetch_assoc() ‚Üí fetch data array
  • $conn->close() ‚Üí prevents connection limits and frees resources
