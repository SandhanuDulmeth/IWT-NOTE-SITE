JavaScript Programming

JavaScript adds interactivity to web pages. Learn variables, functions, DOM manipulation, event handling, and asynchronous programming.

01 Introduction to JavaScript

üìù Lecture Notes

  • JavaScript is a high-level, interpreted programming language
  • Runs in the browser (client-side) and on servers (Node.js)
  • Add to HTML: inline (onclick), internal (<script>), or external file
  • JavaScript is dynamically typed ‚Äî variables don't need type declarations
  • Use console.log() for debugging
javascript 
// Your first JavaScript program
console.log("Hello, World!");

// Variables
let name = "Student";
const course = "IWT";
console.log(`Welcome to ${course}, ${name}`);

02 Variables & Data Types

üìù Lecture Notes

  • let ‚Äî block-scoped, reassignable
  • const ‚Äî block-scoped, cannot be reassigned
  • var ‚Äî function-scoped (avoid in modern JS)
  • Primitive types: String, Number, Boolean, null, undefined, Symbol, BigInt
  • Reference types: Object, Array, Function
  • typeof operator to check types
javascript 
// Primitives
let age = 21;
let name = "Alice";
let isStudent = true;
let grade = null;

// Arrays & Objects
const colors = ["red", "green", "blue"];
const student = {
  name: "Alice",
  age: 21,
  courses: ["IWT", "DSA"]
};

console.log(typeof age); // "number"

03 Functions

üìù Lecture Notes

  • Function declaration: function name() {} ‚Äî hoisted
  • Function expression: const fn = function() {}
  • Arrow function: const fn = () => {} ‚Äî concise, no own this
  • Parameters and default values: function greet(name = "World")
  • Return values with return keyword
  • Scope: local (block/function) vs global
javascript 
// Function declaration
function greet(name) {
  return `Hello, ${name}!`;
}

// Arrow function
const add = (a, b) => a + b;

// Default parameter
const multiply = (x, y = 1) => x * y;

console.log(greet("Alice"));  // "Hello, Alice!"
console.log(add(3, 5));       // 8

04 DOM Manipulation

üìù Lecture Notes

  • DOM = Document Object Model ‚Äî tree representation of the page
  • document.getElementById(), .querySelector(), .querySelectorAll()
  • Modify content: .textContent, .innerHTML
  • Modify styles: .style.property, .classList.add/remove/toggle
  • Create/remove elements: .createElement(), .appendChild(), .remove()
javascript 
// Select elements
const title = document.querySelector('h1');
const items = document.querySelectorAll('.item');

// Modify content & style
title.textContent = 'New Title';
title.style.color = '#E44D26';

// Create & append a new element
const newItem = document.createElement('li');
newItem.textContent = 'New Item';
document.querySelector('ul').appendChild(newItem);

05 Event Handling

üìù Lecture Notes

  • addEventListener(event, handler) is the preferred method
  • Common events: click, submit, keydown, input, mouseover, load
  • The event object contains details (target, type, key, etc.)
  • event.preventDefault() stops default browser behavior
  • Event delegation: listen on parent, check event.target
javascript 
// Click event
document.querySelector('#myBtn')
  .addEventListener('click', (e) => {
    alert('Button clicked!');
  });

// Form submit with preventDefault
document.querySelector('form')
  .addEventListener('submit', (e) => {
    e.preventDefault();
    const data = new FormData(e.target);
    console.log(data.get('name'));
  });

06 Async & Promises

üìù Lecture Notes

  • JavaScript is single-threaded but uses an event loop
  • Callbacks: functions passed as arguments, executed later
  • Promises: represent eventual completion or failure of async operation
  • .then(), .catch(), .finally() for chaining
  • async/await: syntactic sugar over Promises for cleaner code
  • fetch() for making HTTP requests
javascript 
// Fetch API with async/await
async function getUsers() {
  try {
    const res = await fetch('https://jsonplaceholder.typicode.com/users');
    const users = await res.json();
    console.log(users);
  } catch (err) {
    console.error('Error:', err);
  }
}

getUsers();
